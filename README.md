# 贪吃蛇（控制台版）

一个 C++ 控制台贪吃蛇游戏，重点在于**数据结构选择**和**工程化设计**的练手项目。

## 功能

- 经典贪吃蛇玩法（WASD 控制）
- 随机食物生成（使用 `srand(time(0))` 确保每次运行序列不同）
- 撞墙 / 撞自身游戏结束
- 游戏结束后光标恢复（用户体验优化）

## 运行方法

### 编译
```bash
g++ main.cpp -o snake.exe
```

运行

```bash
snake.exe
```

环境要求

· Windows 操作系统
· MinGW / Visual Studio 编译器
· 控制台窗口建议不小于 80×25

技术实现

数据结构

· 蛇身使用 std::deque<std::pair<int, int>>
· 理由：蛇的移动是头部插入、尾部删除，deque 两端操作 O(1)，且内存比链表更连续

渲染

· 使用 windows.h 的 SetConsoleCursorPosition 实现光标定位
· 主动擦除蛇尾（记录上一帧尾巴位置并覆盖空格）
· 无 system("cls") 闪烁问题

随机性

· 使用 srand(time(0)) 播种，确保每次游戏食物位置序列不同

用户体验

· 游戏结束后调用 showCursor() 恢复光标显示

设计决策记录（面试准备）

Q1：为什么用 deque 而不是 vector 或 list？

· vector 头部插入 O(n)，不适合频繁在头部操作
· list 两端操作 O(1)，但缓存不友好，且每个节点多两个指针开销
· deque 两端 O(1)，内存分段连续，最适合贪吃蛇场景

Q2：如何处理蛇尾擦除？

· 在 logic() 中删除蛇尾前，先用 deleteAbletail = snake.back() 记录旧尾巴
· 在 draw() 中对该位置输出空格，避免残影

Q3：为什么不在游戏运行中动态适配窗口？

· 控制台窗口大小改变会导致渲染错位
· 选择启动时检测 + 游戏过程中不处理，明确设计边界
· 这是工程中的需求裁剪，不是技术做不到
