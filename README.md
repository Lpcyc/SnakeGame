# 贪吃蛇（控制台版）设计思路文档
---
## 一、项目目标

- 实现一个控制台贪吃蛇游戏
- 支持可变地图大小
- 支持窗口自适应（启动时）
- 使用C++，仅依赖 Windows 控制台 API（初期）
---
二、地图大小设计

2.1 核心选择

| 问题 | 最终决定 |
|------|----------|
| 固定还是可变 | **可变大小**（用户输入） |
| 正方形还是长方形 | **长方形**（宽、高两个数） |
| 最小地图 | **5×5**（能玩就行） |
| 最大地图 | **由窗口大小决定** |

### 2.2 用户输入处理（防御性编程）

| 输入类型 | 处理方式 |
|----------|----------|
| 非数字 | 提示重新输入 / 使用默认值 |
| 小于最小值 | 强制设为最小值 + 主动提示 |
| 大于窗口限制 | 裁剪到最大值 + 主动提示 |
| 只输入一个数 | 严格要求两个数，否则报错 |

### 2.3 窗口大小 → 最大地图尺寸

```

最大地图宽度 = 窗口宽度 − 4
最大地图高度 = 窗口高度 − 6

```

- `-4`：左右边框 + 边距
- `-6`：上下边框 + 分数行 + 提示行

### 2.4 游戏过程中窗口变化

- **不检测、不允许改变**
- 如果用户强行改变：显示异常或结束游戏（设计约束）

## 三、蛇的数据结构

### 3.1 候选方案对比

| 方案 | 优点 | 缺点 |
|------|------|------|
| 链表 | 理论两端操作快 | 缓存差、代码复杂 |
| 数组（固定） | 简单、快 | 长度固定、浪费或不够 |
| vector | 动态扩容、连续内存 | 头部插入慢 |
| deque | 两端操作快、连续分段 | 比 vector 稍重 |

### 3.2 最终选择deque

理由：vector:头部插入慢，list缓存不友好，多一个指针开销
deque：两端操作O(1),内存比链表更连续

## 四、游戏规则

### 4.1 边界

- 墙是**边界**
- 蛇**不能穿墙**
- 撞墙 = 游戏结束

### 4.2 撞墙判断（逻辑）

假设蛇头坐标 `(hx, hy)`，地图宽度 `w`，高度 `h`：

- 向右：`hx >= w` → 撞墙
- 向左：`hx < 0` → 撞墙
- 向下：`hy >= h` → 撞墙
- 向上：`hy < 0` → 撞墙

### 4.3 食物生成

- 随机生成
- 不能生成在蛇身上
- 当地图快满时：需要检测是否还有空位

### 4.4 胜利条件

- 蛇占满所有格子 → 胜利（可选）

## 五、窗口自适应（启动时）

### 5.1 实现流程

1. 获取控制台窗口大小（`GetConsoleScreenBufferInfo`）
2. 计算最大地图尺寸（公式见 2.3）
3. 提示用户输入地图宽高
4. 校验并裁剪（见 2.2）
5. 初始化游戏

### 5.2 裁剪提示

- **主动提示**：当用户输入被裁剪时，打印提示信息
- 不要求重新输入，直接使用裁剪后的值

### 5.3 不做的功能

- 运行中动态调整地图大小
- 运行中检测窗口变化
